# Search Contracts

`src/search` 负责搜索相关的核心抽象，例如索引接口、查询结果结构和匹配信息。

## 适用范围

- 搜索接口定义
- 索引抽象与实现边界
- 查询结果、匹配片段、评分结构

## 当前代码特征

- `index.ts` 当前以接口/类型为主，定义了 `SearchIndex`、`SearchResult`、`SearchMatch` 等契约。
- 这一层适合作为不同搜索实现的稳定边界，而不是承载 UI 搜索状态。

## 约束

- 搜索接口优先保持实现无关，避免把具体库细节直接写死到对外契约里。
- 返回结果字段要稳定、语义明确，便于 renderer 或其他调用方消费。
- 不要把页面筛选状态、键盘交互、结果高亮 UI 等上层概念放进 core。
- 如果将来引入实际索引实现，保持可替换性和可测试性。
