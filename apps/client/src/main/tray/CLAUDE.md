# System Tray

`src/main/tray` 负责系统托盘图标、托盘菜单以及托盘点击后的窗口可见性行为。

## 适用范围

- Tray 图标路径与 fallback 处理
- 托盘 tooltip 和 context menu
- 点击托盘图标时的窗口显示/隐藏逻辑
- 托盘实例的共享状态保存

## 当前代码特征

- `manager.ts` 统一创建托盘实例并写入 `shared-state`。
- 当前托盘菜单只包含“显示主窗口”和“退出应用”两个核心动作。
- 单击托盘图标时会在隐藏/显示主窗口之间切换。
- 图标加载失败时会退回空图标，避免托盘创建直接报错。

## 约束

- 托盘行为要与窗口关闭到后台的语义保持一致，不要让两者互相打架。
- 与退出相关的操作必须显式维护 `setIsQuiting(true)`，避免误触发“只隐藏不退出”。
- 托盘点击逻辑应保持简洁，不要在这里堆积业务状态判断。
- 平台差异如果变多，优先在这一层集中处理，不要泄漏到更上层入口文件。
